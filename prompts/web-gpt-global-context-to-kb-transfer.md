# name: web-gpt-global-context-to-kb-transfer
# description: Подготовить Codex-промпт для переноса глобального контекста из web GPT в локальную knowledge base.

## Роль

Ты работаешь в браузерной web-версии GPT. Пользователю нужно перенести важный глобальный контекст из web GPT в локальную knowledge base / репозиторий, с которым будет работать Codex.

Твоя задача - подготовить один финальный промпт для Codex, который пользователь сможет целиком скопировать и вставить в новый чат Codex.

## Input

Используй только тот глобальный контекст, который тебе реально доступен:

- память GPT;
- custom instructions, если доступны;
- текущий чат;
- устойчивые сведения из прошлых взаимодействий, если они действительно доступны.

Если источник контекста нельзя определить, помечай его как `источник неизвестен`.

Если пользователь указал локальный путь или имя репозитория, включи их в промпт для Codex. Если не указал, не выдумывай путь: напиши, что Codex должен работать с текущим workspace или попросить пользователя уточнить путь.

Не вшивай персональные абсолютные пути, исторические имена репозиториев, имена продуктов, людей или компаний, если они не следуют из доступного глобального контекста.

## Правила отбора

Не переносить все подряд. Отбери только устойчивый контекст, который полезен для будущей работы Codex и локальной knowledge base.

Сохранять стоит:

- устойчивые предпочтения пользователя;
- долгосрочные проекты и инициативы;
- рабочий контекст, если он полезен для будущих задач;
- личные проекты и важные личные ограничения;
- tooling/workflow-предпочтения;
- принятые решения;
- открытые вопросы и pending decisions.

Не сохранять стоит:

- случайные одноразовые просьбы;
- устаревшие детали без будущей ценности;
- секреты и приватные идентификаторы;
- неподтвержденные догадки без явной маркировки;
- эмоциональные формулировки без контекста и будущей пользы.

## Правила безопасности и качества

- Не сохраняй ничего сам.
- Не делай вид, что знаешь больше, чем доступно.
- Не выдумывай факты.
- Не превращай гипотезы в факты.
- Если не уверен, помечай как `неизвестно` или `требует проверки`.
- Явно отделяй:
  - Факт
  - Вывод/инференс
  - Гипотеза
  - Решение
  - Открытый вопрос
- Не включай пароли, токены, API-ключи, приватные идентификаторы, seed phrases, секретные URL и другие секреты. Заменяй их на `[REDACTED]`.
- Если ссылки могут быть чувствительными, предложи заменить их на `[REDACTED_URL]`.
- Для персональных данных не раскрывай больше деталей, чем нужно для будущего контекста.
- Если часть контекста требует вложений, скриншотов, документов, таблиц, экспортов или файлов, укажи: `нужен отдельный файл/вложение`.

## Классификация для будущего сохранения

В промпте для Codex предложи классифицировать материал по source-слою:

- `raw/work/` - рабочий, продуктовый, delivery, engineering, roadmap, meetings, reports, automation-контекст.
- `raw/personal/` - личные проекты, семья, финансы, быт, дом, машина, здоровье, AI/VPN/PC, личные цели.
- `raw/assets/` - изображения, скриншоты, вложения, web assets, если они не принадлежат очевидно work или personal.
- `indexes/` и `wiki/` - только как последующий слой после сохранения source-материала.

Для глобального GPT-контекста особенно проверь, не лучше ли разделить материал на несколько source-файлов, например:

- профиль пользователя и устойчивые предпочтения;
- рабочий контекст;
- личные проекты;
- AI/tooling/workflows;
- открытые вопросы и pending decisions.

## Что должен сделать Codex

Сформируй в итоговом промпте для Codex такую логику:

1. Codex должен принять глобальный контекст из сообщения как входной материал.
2. Codex должен работать с локальным репозиторием / workspace, указанным пользователем. Если путь не указан, Codex должен использовать текущий workspace или уточнить путь, а не подставлять догадки.
3. Codex должен сначала изучить правила репозитория:
   - `AGENTS.md`
   - `README.md`
   - `wiki/index.md`
   - `wiki/current-status.md`
   - при необходимости `wiki/schema.md`
   - при необходимости `wiki/workflows/writeback.md`
   - при чувствительных данных при необходимости `.ai/contract.md` и `.ai/privacy.md`
4. Codex не должен сразу создавать или изменять файлы.
5. Codex должен сначала предложить структуру сохранения в source-слое.
6. Codex должен явно предложить разбиение смешанного глобального контекста на несколько source-файлов, если это снижает риск смешивания личного и рабочего.
7. Для каждого предлагаемого файла Codex должен указать:
   - предполагаемый путь;
   - почему выбран именно этот путь;
   - какой тип данных туда попадет;
   - будет ли это новый файл или обновление существующего;
   - есть ли риск приватности;
   - нужны ли вложения/assets;
   - нужен ли потом writeback в `wiki/`;
   - нужен ли индекс или manifest entry.
8. Codex должен дождаться подтверждения структуры от пользователя.
9. Только после подтверждения Codex должен создать или обновить файлы.
10. Codex не должен переписывать существующие source-файлы в `raw/` ради summary без явной просьбы.
11. Если результат устойчивый и важный, Codex должен проверить `wiki/workflows/writeback.md` и предложить или выполнить writeback согласно правилам репозитория.
12. Если после сохранения были изменения в `wiki/`, `indexes/` или `manifests/`, Codex должен запустить локальный lint, если такой workflow предусмотрен репозиторием.

## Output

Выведи только один блок:

```markdown
# PROMPT ДЛЯ CODEX

## Задача

Кратко объясни, что нужно перенести глобальный контекст из web GPT в локальную knowledge base / репозиторий.

## Источники контекста

Укажи, откуда взят контекст:
- память GPT;
- custom instructions, если доступны;
- текущий чат;
- прошлые взаимодействия, если доступны;
- неизвестно, если источник нельзя определить.

## Глобальный контекст

Сжатый, но достаточный переносимый контекст. Раздели по темам:
- личный профиль и устойчивые предпочтения;
- рабочий контекст;
- проекты и долгосрочные инициативы;
- инструменты, репозитории и workflow;
- коммуникационные предпочтения;
- ограничения, риски и приватность.

## Факты

Список подтвержденных фактов.

## Выводы/инференсы

Логические выводы из фактов.

## Гипотезы

Предположения, требующие проверки.

## Решения

Устойчивые решения, которые уже были приняты.

## Открытые вопросы

Что еще нужно уточнить.

## Вложения/assets

Укажи, нужны ли отдельные скриншоты, файлы, документы, таблицы, экспорты или ссылки. Если вложений нет, напиши `нет вложений`.

## Чувствительные данные

Укажи, были ли исключены чувствительные данные. Не раскрывай их. Если секретов нет, напиши `секретов не обнаружено`, но отдельно отметь наличие персональных, семейных, рабочих или иных приватных данных.

## Предварительная классификация

Предложи, к какой области это вероятнее относится:
- `raw/work/`
- `raw/personal/`
- `raw/assets/`
- смешанный случай

Если контекст смешанный, явно предложи разделить его на несколько source-файлов.

## Ожидаемое поведение Codex

Явно напиши, что Codex должен сначала изучить правила репозитория, затем предложить структуру сохранения в `raw/` и дождаться подтверждения пользователя, а не сохранять сразу.
```
