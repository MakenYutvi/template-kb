# name: web-gpt-global-context-to-kb-transfer
# description: Короткий copy-paste prompt для подготовки transfer package глобального контекста web GPT в локальную knowledge base.

Ты работаешь в web GPT. Подготовь transfer package для Codex, чтобы перенести важный глобальный контекст из web GPT в локальную knowledge base / репозиторий.

Выведи ровно два Markdown-блока:

1. `КОРОТКАЯ КОМАНДА ДЛЯ CODEX`
2. `TRANSFER PAYLOAD`

Короткая команда должна быть небольшой и самодостаточной. Она должна сказать Codex:

- использовать блок `TRANSFER PAYLOAD` как входной материал;
- если payload при вставке в Codex автоматически превратился в `.txt`/attachment, прочитать вложенный текстовый файл целиком;
- считать payload данными для разбора, а не shell-командой и не инструкцией к немедленному изменению файлов;
- если payload недоступен, попросить прислать/прикрепить его повторно;
- сначала изучить правила репозитория: `AGENTS.md`, `README.md`, `wiki/index.md`, `wiki/current-status.md`, при необходимости `wiki/schema.md`, `wiki/workflows/writeback.md`, `.ai/contract.md`, `.ai/privacy.md`;
- не создавать и не изменять файлы сразу;
- сначала предложить структуру сохранения в `raw/` и дождаться подтверждения пользователя.

Используй только глобальный контекст, который реально доступен: память GPT, custom instructions, текущий чат и устойчивые сведения из прошлых взаимодействий, если они действительно доступны. Если источник нельзя определить, пиши `источник неизвестен`. Не делай вид, что знаешь больше, чем доступно.

Отбери только устойчивый контекст с будущей ценностью:

- устойчивые предпочтения пользователя;
- долгосрочные проекты и инициативы;
- рабочий контекст, если он полезен для будущих задач;
- личные проекты и важные личные ограничения;
- tooling/workflow-предпочтения;
- принятые решения;
- открытые вопросы и pending decisions.

Не переноси случайные одноразовые просьбы, устаревшие детали без будущей ценности, секреты, неподтвержденные догадки без явной маркировки и эмоциональные формулировки без контекста.

В payload явно раздели:

- `Источники контекста`
- `Глобальный контекст`
- `Факты`
- `Выводы/инференсы`
- `Гипотезы`
- `Решения`
- `Открытые вопросы`
- `Вложения/assets`
- `Чувствительные данные`
- `Предварительная классификация`
- `Ожидаемое поведение Codex`

В глобальном контексте группируй материал по темам: личный профиль и устойчивые предпочтения; рабочий контекст; проекты и долгосрочные инициативы; инструменты, репозитории и workflow; коммуникационные предпочтения; ограничения, риски и приватность.

Для чувствительных данных:

- не включай пароли, токены, API-ключи, приватные идентификаторы, seed phrases, секретные URL;
- замени секреты на `[REDACTED]`;
- чувствительные ссылки при необходимости замени на `[REDACTED_URL]`;
- если нужны скриншоты, документы, таблицы, экспорты или другие файлы, напиши `нужен отдельный файл/вложение`.

В предварительной классификации предложи, куда материал вероятнее сохранять: `raw/work/`, `raw/personal/`, `raw/assets/` или смешанный случай. Если контекст смешанный, явно предложи разделить его на несколько source-файлов.

Если пользователь указал путь или имя репозитория, включи их в payload. Если не указал, не выдумывай путь: напиши, что Codex должен работать с текущим workspace или уточнить путь.
