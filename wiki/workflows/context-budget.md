# Context Budget Workflow

Используй этот workflow, чтобы не загружать лишний контекст и не превращать каждый запрос в полный обход репозитория.

## Принцип

Читать нужно минимальный набор файлов, достаточный для надежного ответа.

## Уровни чтения

### Level 0: Direct File

Используй, если пользователь указывает конкретный файл или маленькую папку.

Читать:

- указанный файл;
- ближайшие локальные инструкции, если они есть;
- source-файлы только при необходимости.

### Level 1: Wiki Entry

Используй для восстановления общего контекста.

Читать:

- `AGENTS.md`;
- `wiki/index.md`;
- `wiki/current-status.md`;
- релевантную wiki-страницу.

### Level 2: Source Verification

Используй для точных ответов, решений и правок.

Читать:

- relevant wiki;
- source-файлы из `raw/`;
- decision log, если решение уже принималось.

### Level 3: Lint Or Migration

Используй для структурных изменений, lint и крупных миграций.

Читать:

- `AGENTS.md`;
- `.ai/contract.md`;
- `wiki/index.md`;
- `wiki/schema.md`;
- `wiki/workflows/`;
- `manifests/sources.md`;
- релевантные source-области.

## Правила

- Не читать весь `raw/` без причины.
- Не доверять `wiki/` для точных чисел, дат и юридических формулировок без source verification.
- Если ответ стал полезным устойчивым артефактом, применить `writeback.md`.
- Если найдено противоречие, применить `source-provenance.md`.
